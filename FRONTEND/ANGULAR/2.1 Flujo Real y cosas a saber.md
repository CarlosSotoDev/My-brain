# <mark style="background:#d4b106">Flujo de Datos: Del Usuario al Servidor y de Vuelta</mark>

Este es el camino que recorre la información en una arquitectura profesional:

1. **Interacción en Componente:** El usuario realiza una acción (ej. clic en "Cargar Perfil") en la **Page/Component**.

2. **Delegación al Servicio:** El componente no sabe nada de URLs ni de lógica de red; simplemente invoca un método del **Service** inyectado.

3. **Llamada HTTP:** El servicio utiliza el **HttpClient** para disparar una petición asíncrona hacia el Backend en Java.

4. **Procesamiento en Interceptores:** Antes de salir, un **Interceptor** adjunta automáticamente el token JWT a la cabecera.

5. **Persistencia en Estado:** Al recibir el JSON del Back, el servicio actualiza un **Signal** o un **BehaviorSubject (RxJS)**.

6. **Sincronización de UI:** Gracias a la reactividad de Angular, la vista detecta el cambio en el estado y se repinta automáticamente sin recargar la página.

## <mark style="background:#40a9ff">Anatomía de una Aplicación Angular Senior</mark>

Para dominar el Frontend como un experto en Backend, debes ver estas piezas como capas de una arquitectura limpia:

### <mark style="background:#d3f8b6">1. Componentes (Controller + View)</mark>

Es la unidad básica de la interfaz. Se divide en el archivo `.html` (la estructura visual) y el `.ts` (la lógica de la vista).

- **Mentalidad Senior:** Trátalo como un **Controller**. No debe tener lógica de negocio pesada, solo debe capturar eventos del usuario y decidir qué mostrar basándose en los datos que recibe de los servicios.
    

### 2. <mark style="background:#d3f8b6">Servicios (Business Logic Layer)</mark>

Clases decoradas con `@Injectable`. Son Singletons por defecto.

- **Responsabilidad:** Aquí reside la comunicación con el exterior y la lógica de transformación de datos ligera (formatear fechas, filtrar listas). Es el puente entre el mundo externo (API) y el interno (UI).
    

### 3. <mark style="background:#d3f8b6">HTTP Client (Rest Client)</mark>

Es el equivalente a un _RestTemplate_ o _WebClient_ en Java Spring.

- **Función:** Maneja las peticiones GET, POST, PUT, DELETE. Devuelve un **Observable**, lo que permite manejar flujos de datos en el tiempo, reintentos de conexión y cancelaciones de peticiones de forma elegante.
    

### 4. <mark style="background:#d3f8b6">Estado (Signals o RxJS)</mark>

Es la "Single Source of Truth" (Fuente única de verdad) en el navegador.

- **Signals:** La forma moderna y eficiente de manejar valores que cambian (como el nombre de un usuario).
    
- **RxJS:** Ideal para flujos de datos complejos, como un buscador que filtra mientras escribes o múltiples peticiones en paralelo.
    

### 5. <mark style="background:#d3f8b6">Routing (Navegación)</mark>

Define qué componente se muestra según la URL en la barra de direcciones.

- **Senior Tip:** Se utiliza **Lazy Loading** para que el navegador solo descargue el código del "User Feature" cuando el usuario realmente entra en `/users`, optimizando la carga inicial.
    

### 6. <mark style="background:#d3f8b6">Interceptores (AOP - Aspect Oriented Programming)</mark>

Funciones que capturan cada petición HTTP que sale o entra.

- **Uso Global:** Ideales para inyectar el **JWT** en el header `Authorization` de forma centralizada y para el manejo de errores (ej. si el Back devuelve un 401, el interceptor redirige automáticamente al Login).
