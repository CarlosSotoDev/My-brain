# <mark style="background:#d4b106">De back a Front a 2</mark>
Para un desarrollador con experiencia sólida en Backend, la transición a Angular moderno (versión 17+) es muy natural, ya que el framework comparte principios de ingeniería similares: inyección de dependencias, tipado fuerte y modularidad.

## <mark style="background:#40a9ff">Estructura de Proyecto (Standalone Arquitecture)</mark>

![](ASSETS/IMG/ANGULAR/StandaloneArchitecture.webp)
### **<mark style="background:#d3f8b6">Breve descripción de los niveles:</mark>**

- **Core:** Aquí reside lo que se instancia una sola vez. Es el "motor" invisible de la aplicación.
    
- **Shared:** Contiene piezas que no tienen lógica de negocio propia y pueden usarse en cualquier `feature`.
    
- **Features:** Es la organización por dominio. Cada carpeta es independiente, facilitando que diferentes equipos trabajen en distintas funcionalidades sin colisionar.
---
## <mark style="background:#40a9ff">Cómo pensar esta estructura desde el Backend</mark>

Un error común es intentar aprender Frontend como algo totalmente ajeno. Para un Senior de Backend, es más eficiente "mapear" los conceptos que ya conoce hacia la arquitectura de Angular:

| **Concepto Backend (Java/C#/Node)** | **Equivalente en Angular (Frontend)** | **Descripción del paralelismo**                                                                       |
| ----------------------------------- | ------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| **Controllers / Entry Points**      | **Components (Pages)**                | Es el punto donde el sistema "expone" algo al usuario y recibe sus inputs.                            |
| **Services (Business Logic)**       | **Services (@Injectable)**            | Manejan la lógica de estado y la comunicación con el exterior. Son Singletons inyectables.            |
| **DTOs / Entities / POJOs**         | **Models (Interfaces/Types)**         | Definen la forma de los datos. TypeScript asegura que el contrato del Backend se respete en el Front. |
| **Modules / Packages**              | **Feature Folders**                   | Agrupación por contexto delimitado (Bounded Context) para mantener alta cohesión.                     |
| **Middleware / Filters**            | **Interceptors / Guards**             | Procesan las peticiones antes de salir o las respuestas antes de entrar, y protegen rutas.            |
| **Dependency Injection**            | **Dependency Injection**              | Angular usa el mismo patrón que Spring o .NET para gestionar instancias de servicios.                 |

---

Al diseñar el Frontend bajo este esquema, tratas a cada **Feature** como un "pequeño módulo" de lógica. Esto evita el código espagueti y permite que el proyecto escale de la misma forma que una arquitectura de microservicios bien estructurada. En lugar de pensar en "dibujar la web", piensa en "extender tu arquitectura de servicios hacia el cliente".
