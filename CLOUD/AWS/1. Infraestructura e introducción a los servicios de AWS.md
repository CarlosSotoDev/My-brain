# <mark style="background:#d4b106">1. Infraestructura e introducción a los servicios de AWS</mark>

Los modelos de servicio en la nube representan los distintos niveles de abstracción y control que una organización puede delegar en un proveedor como AWS. Esta clasificación jerárquica (que abarca desde la gestión directa de recursos de infraestructura hasta el consumo de software final) permite a las empresas optimizar si eficiencia operática al definir exactamente que capas tecnológicas desean administrar internamente y cuáles prefiere externaliza para centrarse en el valor centra de su negocio

![]()

--- 
## <mark style="background:#40a9ff">IaaS (Infrastructure as a Service)</mark>

Es el nivel más básico y flexible. AWS te proporciona los recursos de hardware (redes, computadoras virtuales y almacenamiento) y tu te encargas de instalar el sistema operativos, las bases de datos y el código

- **Tu gestionas**: El sistema Operativo (Windows/Linux) , parches de seguridad de la aplicación el "runtime"(Java, Python, etc.)- y los datos
- **AWS Gestiona:** El hardware físico, el cableado, el aire acondicionado del data center y la virtualización
![]()
### <mark style="background:#d3f8b6">Ejemplo en AWS</mark>
- **Amazon ECS (Elastic Compute Cloud)**: Proporciona capacidad de cómputa segura y ajustable. Es el equivalente a tener servidores físicos donde tú decides qué sistema operativo y software instalar
- **Amazon S3 (Simple Storage Service)** Ofrece almacenamiento de objetos escalable. Aunque parece un servicio simple, funciona como IaaS porque tú gestionas la estructura de tus datos, permisos de acceso y políticas de retención de archivos
- **Amazon VPC (Virtual Private Cloud)** Te permite aprovisionar una sección de la nube de AWS lógicamente aislada, donde puedes lanzar recursos en una red virtual definida por ti (IPs, subredes y tablas de ruteo)

---
## <mark style="background:#40a9ff">PaaS (Platform as a Service)</mark>

Aquí, AWS elimina la necesidad de gestionar el sistema operativo y el hardware. Tu solo te preocupas por desplegar y administrar tus aplicaciones. Es ideal para desarrolladores que no quieren perder tiempo configurando servidores

- **Tu gestionas**: Solo tu código y la configuración de la aplicación
- **AWS Gestiona:** El sistema operativo, el middleware, el servidor web y el escalamiento automático

### <mark style="background:#d3f8b6">Ejemplo en AWS</mark>
- **AWS Elastic Beanstalk**: Es un servicio de fácil uso para implementar y escalar aplicaciones web desarrolladas en Java, .NET, PHP, Node.js, entré otros. Solo subes el código y Beanstalk maneja el resto
- **AWS Lambda**: Es la definición de la computación serverless. Permite ejecutar fragmentos de código en respuesta a eventos sin administrar servidores; AWS se encarga de que el entorno de ejecución esté listo en milisegundos
- **Amazon RDS (Relational Database Service)**: Facilita la configuración, el funcionamiento y el escalado de bases de datos relacionales (MySQL, PostgreSQL, Oracle). AWS gestiona las copias de seguridad y las actualizaciones del motor, mientras tú gestionas los datos y esquemas
  
  ![]()

## <mark style="background:#40a9ff">SaaS (Software as a Service)</mark>

Es un producto completo que es gestionado directamente por el proveedor. No tienes que preocuparte por como se mantiene el servicio o cómo se gestiona la infraestructura; solo lo usas a través de un navegador o una API

- **Tu gestionas**: Solo la configuración del usuario y tus propios datos dentro de la plataforma
- **AWS Gestiona:** Todo, desde el hardware hasta el mantenimiento del software y las actualizaciones de la aplicación

### <mark style="background:#d3f8b6">Ejemplo en AWS</mark>
- **Amazon QuickSight**: Un servicio de inteligencia de negocios (BI) totalmente gestionado que te permite crear y publicar paneles interactivos de visualización de datos de forma inmediata.
- **Amazon Connect**. Un centro de contacto (Contact Center) omnicanal basado en la nube. Es un software completo que las empresas usan para atención al cliente sin configurar centrales telefónicos físicas.
- **Amazon GuardDuty**: Un servicio de detección de amenazas inteligente que monitorea continuamente tus cuentas de AWS en busca de actividad maliciosa, entregando resultados directamente en una consola lista para usar.

## <mark style="background:#40a9ff">Resumen</mark>

- **Modelos de Servicio:** La computación en la nube se divide en tres niveles de abstracción (**IaaS, PaaS y SaaS**) que definen el modelo de responsabilidad compartida. Cada nivel determina qué parte de la infraestructura gestiona AWS y qué parte recae sobre el usuario.
    
- **Infraestructura y Plataforma:** Mientras que **IaaS** (como **EC2, S3 y VPC**) ofrece control total sobre el hardware virtual y redes, **PaaS** (como **Elastic Beanstalk, Lambda y RDS**) agiliza el desarrollo al automatizar la gestión de sistemas operativos y entornos de ejecución.
    
- **Software como Servicio:** El modelo **SaaS** (como **QuickSight, Connect y GuardDuty**) entrega soluciones finales listas para su consumo. En este nivel, el usuario se enfoca exclusivamente en la interacción con el software, delegando toda la complejidad técnica y el mantenimiento a AWS.
